# Partner Link Bridge — `plink:` schema

AgentHQ (the agent workspace) and the MS Lending demo (the loan-officer workspace) deploy to
the **same GitHub Pages origin** (`buildingresourcesms-byte.github.io`), so they share
`localStorage`. That shared storage *is* the integration — no backend, no accounts, no API.

Each app reads and writes namespaced `plink:` keys and listens to the `storage` event, so two
open tabs (one per app) update each other **live**.

## Keys

### `plink:link:julene-holly`

The handshake itself. Written by whichever side connects first; AgentHQ seeds it on first run
because the Holly ↔ Julene pair ships pre-connected.

```json
{ "status": "connected", "since": "2026-06-12T00:00:00.000Z" }
```

### `plink:referrals`

A JSON **array**. AgentHQ **appends** one object every time Holly sends a buyer to Julene
(one-tap referral, Financing tab, or a client card). The LO app **reads** this list and turns
new entries into leads.

```json
[
  {
    "id": "ref-abc123",
    "from": "holly",
    "name": "Marcus & Tina Boyd",
    "phone": "(601) 555-0233",
    "loanType": "Conventional",
    "purpose": "Purchase",
    "sentAt": "2026-06-12T15:13:03.821Z"
  }
]
```

- `id` — unique, generated by the writer (`ref-` + base36 timestamp + random suffix)
- `from` — agent id (currently always `"holly"`)
- `loanType` — `Conventional | FHA | VA | Jumbo`
- `purpose` — `Purchase | Refinance`

Writers must **append**, never replace, so neither side loses the other's history.

### `plink:status:<buyerKey>`

Live loan status, **written by the LO app** whenever a shared buyer's loan moves, **read by
AgentHQ** for the live financing panels. AgentHQ falls back to its seed data when a key is
absent.

```json
{ "stage": "Underwriting", "estClosing": "2026-07-16", "updatedAt": "2026-06-12T15:00:00.000Z" }
```

- `stage` — one of the shared portal stages, exactly:
  `Application Started · Documents Submitted · Loan Review · Underwriting · Final Approval · Closing`
- `estClosing` — `YYYY-MM-DD`
- Shared buyer keys: **`carter`**, **`ellis`**, **`lawson`**
  (Devin & Alyssa Carter, Jordan & Maya Ellis, Tom & Erica Lawson — the three deals seeded in
  both demos)

## Live updates

Both apps should subscribe:

```js
window.addEventListener('storage', (e) => {
  if (e.key?.startsWith('plink:')) {
    /* re-read the key and update UI */
  }
})
```

The `storage` event only fires in *other* tabs on the same origin — which is exactly the demo:
open AgentHQ and the LO app side by side, advance a loan in the LO app, and watch Holly's
financing panel and milestone feed update without a refresh.

## Adoption checklist for the LO repo (`ms-lending-demo`)

1. On loan status change for a borrower with `buyerKey` in {carter, ellis, lawson}:
   write `plink:status:<buyerKey>` with the portal-stage label.
2. Poll/subscribe to `plink:referrals`; create a New Lead for each unseen `id`
   (track seen ids in `msl-seen-referrals` or similar — outside the `plink:` namespace).
3. Read `plink:link:julene-holly` to render the connected state on the Partners page.

*All names, deals, and data are fictional demo data.*
